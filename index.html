User Manual & Technical Reference

1. Overview

This software optimizes the vertical profile (rail levels) of a railway line. The goals are: (a) maximize smoothness to ensure high-speed riding comfort, (b) minimize lifts/lowers (deviation from the existing rail level) to facilitate maintenance on a running track, (c) ensure that vertical versines and block-wise standard deviations stay strictly within safety limits.

2. Versine Calculation (Vertical Unevenness)

The vertical versine at station $i$ on a chord of length $L$ is calculated as the vertical offset of the rail level at station $i$ relative to the chord drawn between stations $(i - k)$ and $(i + k)$, where $2k \times \text{Interval} = L$.

  • 20m Chord: Evaluated using station $i$ as midpoint, and $(i - 1)$ and $(i + 1)$ as endpoints (for 10m intervals, $k = 1$). $V_{20} = P_i - \frac{P_{i-1} + P_{i+1}}{2}$. Max limit: 5.0 mm.
  • 80m Chord: Evaluated using station $i$ as midpoint, and $(i - 4)$ and $(i + 4)$ as endpoints (for 10m intervals, $k = 4$). $V_{80} = P_i - \frac{P_{i-4} + P_{i+4}}{2}$. Max limit: 40.0 mm.

3. Standard Deviation (SD) in 200m Blocks

Standard deviation is calculated over a sliding window of 200 meters. For 10m station intervals, this represents a block of 20 intervals (21 stations). The standard deviation of the versines calculated within this block must satisfy:
• 20m Chord SD: ≤ 2.0 mm.
• 80m Chord SD: ≤ 20.0 mm.

4. Boundary Conditions

The endpoints (first and last stations) are locked to the existing track level (lift/lower = 0) to ensure a smooth, hardware-compatible transition to the existing track. For 80m chords, the first 4 and last 4 stations cannot have their 80m versine calculated fully (due to chord endpoints extending beyond the data set). These boundary chord limits are handled gracefully by omitting/clamping the outer nodes from SD calculations.

5. How to Use

  1. Specify the start/end chainage and click Generate Grid to enter data manually, or click Template to download a blank Excel template.
  2. Import your existing rail levels using Import Excel.
  3. Adjust constraints (max lift, max lower, versine limits) as desired.
  4. Click Run Auto-Smoother. The solver will run a Projected Gradient Descent with Adam Optimizer to find the smoothest profile that complies with all limits.
  5. If necessary, manually lock specific station levels or edit the Proposed levels in the table. The profile chart, lifts, versines, and compliance stats will update instantly!
  6. Export a detailed PDF engineering report or Excel spreadsheet.

6. Handling Short & Long KMs

The optimization engine calculates physical distances strictly based on the sequential row intervals imported, rather than calculating distance from the chainage text itself. To handle short or long KMs (e.g. 500.980 jumping directly to 501.000), simply ensure your stations in the imported Excel file are listed sequentially and represent an equal physical distance (e.g. 10m) between each row. Avoid using the "Generate Grid" function for these specific stretches as it assumes a perfect 1000m length per Km.
